# 먹어볼래 (Mukabolrae) 프로젝트 발표 자료

## 1. 작품 요약 (배경 및 목적)

### 배경
- 현재 맛집을 찾을 때 네이버 블로그를 많이 이용하지만, **광고성 블로그 포스팅**이 급증하고 있어 신뢰할 수 있는 정보를 찾기 어려운 상황
- 대학생들이 주변 맛집을 찾을 때 **광고인지 실제 후기인지 구분하기 어려워** 잘못된 선택을 하는 경우가 빈번
- 기존 맛집 검색 서비스들은 단순한 평점 제공에 그쳐 **심층적인 분석과 신뢰도 검증** 기능이 부족

### 목적
- **AI 기반 광고 판별 시스템**을 통해 신뢰할 수 있는 맛집 정보만 제공
- **다양한 리뷰 데이터를 분석**하여 객관적이고 종합적인 맛집 평가 제공
- 대구대학교 하양캠퍼스 주변 맛집에 특화된 **로컬 맛집 추천 플랫폼** 구축

## 2. 작품 소개

### 개념 및 설계
- **웹 기반 맛집 검색 및 분석 플랫폼**
- 사용자가 맛집을 검색하면 **실시간으로 블로그를 크롤링**하여 광고 여부를 판별
- **AI 모델과 규칙 기반 시스템**을 결합한 하이브리드 광고 판별 방식 적용
- **종합적인 감성 분석**을 통해 맛집의 실제 평가를 제공

### 작품 특징
1. **AI 광고 판별 시스템**: KoBERT 모델을 활용한 블로그 광고 판별 (70-90% 정확도)
2. **실시간 크롤링**: 네이버 블로그 API와 BeautifulSoup를 이용한 실시간 데이터 수집
3. **OCR 기반 텍스트 분석**: 이미지 내 텍스트까지 분석하여 광고 판별 정확도 향상
4. **KoBART 리뷰 요약**: 긍정/부정 리뷰를 자동으로 분류하고 요약 제공
5. **감성 분석 및 해시태그**: 맛집별 감성 점수와 대표 키워드 제공
6. **사용자 피드백 시스템**: 사용자의 평가를 통한 지속적인 시스템 개선

### 사용 기술 및 작동 원리

#### 백엔드 (Python)
- **Framework**: FastAPI
- **AI 모델**: 
  - KoBERT (광고 판별)
  - KoBART (텍스트 요약)
  - Tesseract OCR (이미지 텍스트 추출)
- **데이터 처리**: pandas, torch, transformers
- **웹 크롤링**: requests, aiohttp, BeautifulSoup4
- **텍스트 분석**: kiwipiepy, rapidfuzz

#### 프론트엔드 (React)
- **Framework**: React 18.3.1
- **UI 라이브러리**: React Bootstrap, FontAwesome
- **차트**: Chart.js, react-chartjs-2
- **라우팅**: React Router DOM

#### 작동 원리
1. **검색 단계**: 사용자가 맛집명 입력
2. **데이터 수집**: 네이버 블로그 API로 관련 블로그 검색
3. **크롤링**: 비동기 방식으로 블로그 내용 수집
4. **AI 분석**: 
   - 텍스트 기반 광고 판별
   - OCR로 이미지 내 텍스트 분석
   - 규칙 기반 패턴 매칭
5. **결과 제공**: 광고 확률, 감성 분석, 리뷰 요약 등 종합 정보 제공

## 3. 결과물

### 주요 화면
1. **메인 검색 화면**: 깔끔한 검색 인터페이스
2. **검색 결과 화면**: 블로그별 광고 확률과 추천도 표시
3. **맛집 상세 정보**: 감성 분석, 해시태그, AI 요약 제공
4. **피드백 대시보드**: 사용자 피드백 통계 및 분석

### 데이터베이스
- **맛집 정보**: 대구대 하양캠퍼스 주변 100여개 맛집 데이터
- **리뷰 데이터**: 네이버 플레이스 리뷰 데이터 수집 및 전처리
- **감성 분석**: 맛집별 긍정/부정 해시태그 및 감성 점수

## 4. 기대효과

### 사용자 측면
- **신뢰할 수 있는 맛집 정보** 획득으로 시행착오 비용 절약
- **객관적이고 종합적인 평가**를 통한 합리적 선택 지원
- **시간 절약**: AI 기반 자동 분석으로 빠른 의사결정

### 기술적 측면
- **AI 모델의 실용화**: 학술적 연구를 실제 서비스에 적용
- **하이브리드 분석 방법론**: 규칙 기반 + AI 모델의 효과적 결합
- **실시간 데이터 처리**: 대용량 크롤링 데이터의 효율적 처리

### 사회적 측면
- **허위 광고 방지**: 광고성 포스팅에 대한 경각심 제고
- **소비자 권익 보호**: 정보의 투명성 확보
- **로컬 상권 활성화**: 진정한 맛집 발굴을 통한 지역 경제 기여

### 확장 가능성
- **다른 지역으로 확장**: 전국 단위 서비스로 발전 가능
- **다양한 업종 적용**: 카페, 숙박업소 등 타 분야 확장
- **상업적 활용**: 맛집 추천 앱, 리뷰 플랫폼 등으로 발전
